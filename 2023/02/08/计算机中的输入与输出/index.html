以LC3为例

I/O

计算机必然是需要与人进行交互的，最基础的交互就是input and output

Privilege, Priority, Memory Address Space

这是一些需要了解的计算机系统的名词

Privilege

我也不知道怎么翻译，所以直接称为privilege了。privilege指的是做一些事情的权利。在计算机中，我们必然是不希望所有的程序都有执行所有ISA中的指令的权利。比如HALT指令可以终止程序的运行，如果有恶意程序的使用HALT会导致计算机运行奔溃。因此需要对其做出区分：

  • privileged ----> supervisor mode
  • unprivileged ----> user mode

前者有超级权限，可以使用所有的指令。

Priority

优先级。计算机遇到多个程序时会根据优先级的大小做出反应

Memory address space

内存地址空间并不是全部可以让用户使用。内存实际上被划分为了多个区域，承担不同的功能。有的部分与系统运行有关，一般情况下用户无法使用。从x0000到xffff依次连续划分为如下：

  • system space
  • supervisor stack
  • user space
  • user stack
  • I/O page

其中前两个区域被一起称为 privileged memory

Input and Output

所有的输入输出设备都是通过寄存器来处理输入输出的数据的。即使最简单的输入输出设备都至少有2个寄存器：

  • 一个用来存储设备与计算机之间交互的信息
  • 一个用来表示设备的状态

在谈输入输出时我们需要了解设备的如下特性：

  • ISA是否需要额外的特殊的指令来处理I/O
  • I/O设备和计算机保持同样的速度吗？如果不是，什么来manage它们之间速度的不同
  • 谁来控制计算机和设备之间的信息传输

下面分别对应这3个问题的解答

memory-mapped I/O vs special I/O instructions

后者是针对输入输出专门定制指令，现在几乎没人用了，现在大部分用的是memory-mapped。简单说，就是把输入输出设备的寄存器视为内存中的一个特定的地址，原先的所有指令都不用变。LC3用的就是这种内存映射方法。地址中的xFE00到xFFFF就是专门为输入输出设备的寄存器保留的。

asynchronous vs. synchronous

同步还是异步？ 同步不太现实，因为处理器的频率是极高的，现实中的输入和输出要比CPU满的多。尽管我们可以通过调整实现同步（即，几百万个时钟周期对应一次输入），但是因为我们的输入并不是匀速的，不能保证输入频率始终与其匹配。 所以实际上用的比较多的还是异步方法。这个的逻辑很简单，就是使用一个状态码flag。通过这个flag可以实现异步输入输出与CPU的同步。以输入为例，每次有新字符输入就set flag，每次有新字符被读取就clear flag。如此一来只需要检测flag的状态就可以判断是否发出对应指令去读取设备的寄存器。

interrupt-driven vs. polling

就是谁来控制设备与计算机之间交互的问题。 所谓的interrupt-driven就是处理器一直在干自己的事，直到设备发出信息：有新的字符输入，请读取。这就是由输入输出设备主导的交互。 polling就是处理器不停地、一遍又一遍地查询状态码-the ready bit，直到它被set。

从键盘输入

basic input registers(KBDR and KBSR)

键盘的两个寄存器分别为 keyboard data register(KBDR)和keyboard status register(KBSR)。它们都在内存中有特定的地址：

  • KBDR: xFE02
  • KBSR: xFE00

LC3内存的字长为16bit，但是这两个寄存器都用不到全部的16bit。对KBDR来说，bit[15:8]不用，总是为0；bit[7:0]为该字符的ASCII码值。 对于KDSR来说，只有bit[15]保存有ready bit，其余位都没用。

the basic input service routine

流程就是，如果由计算机处理器去主导交互，则处理器会一遍又一一遍地查询KBSR直到KBSR[15]被set。当在键盘上打出一个字时，KBSR[15]就被set，对应的字符的ASCII码会被加载进寄存器KBDR，然后处理器会将KBDR中的值load进计算机的一个寄存器中，再把KBSR[15]清空。代码如下所示：
