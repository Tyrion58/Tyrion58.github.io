Event_camera（1）---背景

event camera的原理

标准相机：完全依照外部时钟指定的固定速率（例如30fps）来获取完整图像。 事件相机：对场景中的亮度变化，每个像素独立地，异步地做出反应。因此事件相机的输出是可变的数据速率序列。“事件”就是指特定像素在特定时间亮度的改变。 每次像素发出一个“事件”后，它会记录当前的log intensity，然后继续监控亮度的变化。当亮度的改变程度达到一定的阈值后，就会发送出“event”。x和y代表像素的位置，t代表时间的发生时间，p代表时间的极性（正事件还是负时间）。 事件相机是数据驱动型的传感器,其数据量取决于场景中的动态和光线变化情况。移动越快，其数据采集越快。 ### 事件相机的带宽 当像素点的光强变化过快时，一些事件可能会被当作噪声过滤掉，造成检测到的事件个数减少。这个截止的频率是光照强度的单调递增函数。

event camera designs

DVS 事件相机起源于基于帧的硅视网膜设计，其中连续时间光感受器电容耦合到读出电路，每次采样像素时该读出电路都会重置。 电子和神经科学文献对最近的事件相机技术进行了综述。 尽管令人惊讶的是，许多应用程序仅通过处理 DVS 事件（即亮度变化）就可以解决，但很明显，有些应用程序还需要某种形式的静态输出（即“绝对”亮度）。 为了解决这个缺点，已经开发了几种同时输出动态和静态信息的相机。 基于异步时间的图像传感器 (ATIS)具有包含 DVS 子像素（称为变化检测 CD）的像素，该子像素触发另一个子像素以读出绝对强度（曝光测量 EM）。

优点

  • 高时间分辨率
  • 低延迟
  • 低能耗
  • 高动态范围

新传感范式带来的挑战

  • 应对不同的时空输出。基于传统图像的算法不能直接应用于事件相机的输出。
  • 应对不同的光度感知：与标准相机提供的灰度信息相比，每个事件都包含二进制（增加/减少）亮度变化信息。 亮度变化不仅取决于场景亮度，还取决于场景与相机之间当前和过去的相对运动。
  • 应对噪声和动态效应。

event process

事件相机带来的一个关键问题是，如何从事件相机的数据中提取有效信息来满足当前的任务。目前的算法可以分为两类： - events-by-events bias。系统的状态依据单个事件的到达而发生改变，这意味着极低的延迟。 - operate on groups or packets events。这必然会带来一些延迟，

因此，这两个类别之间的区别更加微妙：单独的事件并不能为估计提供足够的信息，因此需要过去事件或额外知识形式的附加信息。

根据事件是如何被处理的，可以把方式分为： - model-based approaches - model-free(例如数据驱动，机器学习) 或者考虑事物或损失函数的类型： - geometric- - temporal- - photometric-based（例如事件极性或事件活动的函数）

event representations

事件经常被处理和翻译成representations，便于提取出有意义的信息来解决问题。

  • individual events \(e_k=(x_k, t_k, p_k)\)。常常被用于event-by-event处理方式。例如probabilistic filters, Spiking Neural Networks。
  • event packet。在时空上相邻的一批事件被一起处理，得到一个输出。常常根据任务的不同，选择合适的packet尺寸
  • event frame/image or 2D histogram。在时空上相邻的事件按照相同的方式（例如统计事件的个数，或者极性）转化成一个图像（2D grid），这个图像可以用计算机视觉算法处理。尽管这些图像是和自然图像不同的，但是一些算法仍然有效。然而，这种做法在基于事件的范式中并不理想，因为它量化了事件时间戳，可以丢弃稀疏性，并且生成的图像对使用的事件数量高度敏感。
  • Time surface(TS)。ts是一个2D图，每个像素点保存着一个单独的时间值（例如在这个点发生的上个事件的时间戳）。
  • voxel grid.什么是voxel？Google翻译的解释是：（在基于计算机的建模或图形模拟中）构成概念三维空间的体积元素阵列中的每一个，尤其是三维对象的表示被划分成的离散元素阵列中的每一个。
  • 3D point set。在时空上相邻的事件在三维空间中被当作点来处理。这是一种稀疏表达，经常被用于point-based geometric processing methods，例如:plane fitting, pointNet
  • point sets on image plane：事件被视为图像平面上一组不断演变的二维点
  • motion-compensated event image：不仅依赖于事件而且依赖于移动假设。motion compensation：当一个边在图层上移动时，它在它所通过的像素点上激发出事件；这种representaion表明一种衡量events有多契合候选移动
  • reconstructed images：通过图像重建得到的亮度图可以提供一种更运动不变的表示（相比于TS和event frames）。

event polarity：事件极性的处理有两种方法： - 分别处理positive和negative的事件，将结果合并 - 用一种一般的方式一起处理他们

methods for event processing

event-by-event-based methods

首先是确定性过滤器（deterministic filters），例如（时空）卷积，活动过滤器被用于降噪和特征提取、图像重构、亮度过滤。概率性过滤器（probabilistic filters）常用在SLAM系统中。 filters是event-by-event processing的主流方法，因为： - 处理异步数据，因此能提供最小的处理延迟 - 将来自多个小源的信息整合到一起

另一个主流方法是多层ANN，网络含有很多参数，这些参数需要从事件数据来计算。用典型的无监督学习训练的网络作为的用于分类的特征提取器（如SVM），仍然需要一些有标签数据用于训练。如果有足够的标签数据，反向传播之类的监督学习方法可以不需要单独的分类器地去训练网络。许多方法使用事件的packets去训练，之后把训练好的网络转换成SNN来event-by-event地处理数据。 ## methods for groups of events 单个事件能提供的信息很少，处理多个事件组合成的groups可以减少噪声的干扰。representation不同，处理数据的方法也会有所不同。 许多方法是通过处理数据，使其能够使用已有的计算机视觉工具。在这个层面，event frames是一个典型的表示方式。 ## Biologically inspired visual processing 生物原理驱动了事件相机像素点处理的设计和算法。例如spiking nerual networks(SNNs)。 ### visual pathways DVS仿照生物的可视化机制。在生物成像机制中有瞬时通路。
